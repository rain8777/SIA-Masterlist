📖 Setup & Integration Guide

Step 1 — Create Google Sheet

Create a Google Sheet named SIA_Masterlist with these exact column headers in Row 1:

id | familyName | givenName | middleName | dob | gender | purok | barangay
motherFamily | motherGiven | motherMiddle | philhealth | contact | relationship
vaccine | dateVacc | vaccSite | status | vaccinatorFamily | vaccinatorGiven
vaccinatorMiddle | designation | remarks | dateEncoded | encodedBy

Also create a second sheet named Users for user management (columns: id, username, passwordHash, role, name, active).

Step 2 — Apps Script Backend

Go to Extensions → Apps Script and paste the code from appscript/Code.gs (included in this package). Then deploy as:

Deploy → New Deployment → Web App → Execute as: Me → Who has access: Anyone
Copy the Web App URL and paste it in ⚙️ Config.

Step 3 — GitHub + Vercel Deployment

Push all files to a GitHub repo. Connect to Vercel — it will auto-deploy on every push. The vercel.json configures all security headers (HSTS, CSP, X-Frame-Options, etc.).

Security Best Practices

  • Remove the demo credentials box in login.html before deploying to production
  • Enable Google Account authentication on the Apps Script for extra protection
  • Limit Apps Script sharing to specific Google accounts/domains
  • Enable 2-Factor Authentication on the Google account owning the Sheet
  • Regularly export and back up the Google Sheet
  • Change default passwords immediately after first login
  • Comply with DOH Data Privacy and RA 10173 requirements

Google Apps Script Backup (GAS Fallback)

In your Apps Script project, add a new HTML file named index and paste the contents of index.html. Update your doGet() to serve it:

function doGet(e) {
  if (!e.parameter.action) {
    return HtmlService.createHtmlOutputFromFile('index')
      .setTitle('SIA Masterlist')
      .setSandboxMode(HtmlService.SandboxMode.IFRAME);
  }
  // ... handle API actions below
}
